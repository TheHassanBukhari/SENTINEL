Graphs, BFS, Dijkstra, and Prim's Algorithm

The three most powerful tools in SENTINEL. All three operate on graphs. Understanding the graph first makes everything else click.

1. What Is a Graph

Nodes connected by edges

A graph is the most general way to represent relationships between things. It has two components:

  • Nodes (also called vertices): the things. A person, a city, a router, a webpage.
  • Edges: the connections between things. A friendship, a road, a cable, a hyperlink.

That is all a graph is. Dots and lines between them. But this simple idea can model almost any real world system.

A weighted graph. Five nodes, six edges, each edge has a weight representing cost, distance, or risk.

Weighted vs unweighted

An unweighted graph just says whether two nodes are connected or not. A weighted graph assigns a number to each edge. That number can mean distance, cost, time, risk, or anything meaningful to the problem.

Directed vs undirected

An undirected graph means the connection goes both ways. If A is connected to B then B is connected to A. A directed graph means connections have a direction. A can point to B without B pointing back to A. Like a one way street.

How a graph is stored in code

Each node is an object. Each node has a linked list of its neighbors called an adjacency list. Each item in that list contains the neighbor's ID and the edge weight between them. The whole graph is a collection of these node objects.

Everything in the real world that involves relationships can be modeled as a graph. Social networks, road maps, the internet, disease spread, power grids. The three algorithms below are how you extract meaning from a graph.

2. BFS: Breadth First Search

Explores level by level outward from a starting node

BFS answers the question: starting from one node, in what order do you reach all other nodes if you always explore the closest ones first?

Imagine dropping a stone in still water. Ripples spread outward in rings. The first ring is closest to the stone. The second ring is one step further. The third ring is two steps out. BFS explores a graph exactly like this, ring by ring, level by level.

How BFS works step by step

1
Start at the source node. Mark it as visited. Add it to the queue.
2
Dequeue the front node. Look at all its unvisited neighbors. Mark each as visited and add each to the back of the queue.
3
Repeat. Dequeue the next node. Visit its unvisited neighbors. Add them to the queue.
4
Continue until the queue is empty. Every node reachable from the source has been visited.

BFS from node A. First it visits A. Then B and C (one step away). Then D, E, F, G (two steps away). It always finishes one full wave before starting the next.

Why BFS needs a Queue

The Queue is what enforces the wave by wave order. When you discover neighbors of A you add them to the back of the queue. B and C are now in the queue. When you process B you add its neighbors D and E to the back. But C is still at the front ahead of D and E so C gets processed before them. This naturally produces the wave order.

Without a queue, BFS does not work. The FIFO property of the queue is precisely what keeps exploration in rings rather than diving deep into one branch.

3. Dijkstra's Algorithm

Finds the shortest weighted path between two nodes

BFS finds the path with the fewest hops. But what if edges have weights and you want the path with the lowest total weight, not the fewest edges?

Think of a road map. You want to drive from city A to city D. There might be a route with only two roads but one of them is 800 km long. There might be another route with four roads that totals only 320 km. BFS would pick the two road route. Dijkstra picks the 320 km route because it finds the minimum total cost.

The core idea

Assign every node a distance value. The source node gets distance zero. Every other node gets infinity meaning unknown so far. Then repeatedly do this: pick the unvisited node with the smallest current distance, visit it, and update all its neighbors. If going through the current node gives a neighbor a shorter total distance than what it had before, update the neighbor's distance. Keep going until you reach the destination.

Step by step with an example

Dijkstra finds A to D costs 5 via C, not 9 via B, and not 9 via the direct route. It always finds the true minimum.

StartASet A=0, all others=∞A=0, B=∞, C=∞, D=∞

The guarantee: Dijkstra never updates a distance to something worse. It only improves distances. This means when a node is marked visited, its distance is final and correct. It will never be found to be wrong later.

4. Prim's Algorithm

Builds the minimum spanning tree of a graph

Dijkstra answers: what is the cheapest path from A to B? Prim's answers a different question entirely: what is the minimum set of edges that keeps every node in the graph connected to every other node?

This result is called a Minimum Spanning Tree, or MST. A spanning tree connects all nodes with no cycles, using the minimum total edge weight.

The analogy

Imagine you have 8 villages and you need to lay electrical cable so every village has power. You do not need every village directly connected to every other village. You just need every village reachable through some path. And you want to use the least total cable length.

Prim's finds exactly which connections to make.

How Prim's works

1
Start at any node. Add it to the MST set.
2
Look at all edges that connect a node inside the MST set to a node outside it. Pick the one with the lowest weight.
3
Add that edge and its new node to the MST set.
4
Repeat steps 2 and 3 until all nodes are in the MST set.

Prim's selects only 3 edges from the 6 available. All 4 nodes are still connected. Total cost is 12, the minimum possible.

Key difference from Dijkstra

DijkstraPrim's
Question answeredShortest path from A to BMinimum edges to connect all nodes
OutputA single pathA tree connecting everything
Use caseNavigation, routingNetwork design, infrastructure
Cares aboutTwo specific nodesThe entire graph
Common confusion: Both algorithms look similar in code. Both pick the minimum weight at each step. The difference is what they are minimizing. Dijkstra minimizes the distance from the source to each node. Prim's minimizes the weight of the edge being added to the growing tree. Different goals, different outputs.
