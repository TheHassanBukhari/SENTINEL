Foundations: Memory, Nodes, and Linked Lists

Before understanding any data structure, you need to understand how data lives in memory. This part builds that base from scratch.

1. How Memory Works

Your computer's RAM is like a very long street with millions of numbered houses. Each house holds one small piece of data. The house number is called an address.

When your program needs to store something, the operating system hands it some houses to use. The question is: which houses, and how do you find your data again later?

RAM is a long row of numbered slots. Each slot holds data. Your program accesses data by its address.

2. Arrays: The Simple Way

The simplest way to store a list of things is an array. You ask for 5 boxes in a row, the OS gives you 5 consecutive addresses, and you use index numbers to find each item.

An array: five consecutive boxes. Access any element instantly using its index number.

The problem with arrays: You must decide the size upfront. If you say "give me 5 boxes" and later need a 6th, you cannot just add one. The next house in memory might already be occupied by something else. You would have to move everything to a bigger empty street. This is slow and wasteful.

3. Pointers: The Key Idea

A pointer is a variable that does not hold data. It holds an address. It points to where the actual data lives.

Think of it like this. Instead of writing Ahmed's phone number on a piece of paper, you write "Ahmed's number is in the red notebook on page 4." That piece of paper is a pointer. It does not contain the phone number, it tells you where to find it.

A pointer holds an address. Follow the address and you reach the real data.

In C++, you write a pointer like this: int* ptr = &someVariable;. The * means "this is a pointer." The & means "give me the address of."

4. Nodes: Data Plus a Pointer

Now combine the two ideas. What if each piece of data also held a pointer to the next piece of data? You would not need consecutive memory at all. Each item knows where the next item is.

This combined thing, data plus a pointer, is called a node. It is the single most important building block in all of the data structures in SENTINEL.

A node is data and a pointer living together. The pointer says where the next node is.

In C++ a node looks like this:

struct Node {
  string name;
  int age;
  Node* next;
};

Node* next is a pointer to another Node. That is all it takes. This one idea is the foundation of everything.

5. Linked List: Chaining Nodes Together

A linked list is simply a chain of nodes. The first node points to the second. The second points to the third. The last one points to nullptr, which means nothing, end of the list.

You only need to remember the first node. From there you can reach every other node by following the chain of pointers.

A linked list. HEAD points to the first node. Each node points to the next. Last node points to null meaning the list ends here.

Why linked lists beat arrays here: Nodes can live anywhere in memory. No need for consecutive slots. Adding a new patient just means creating one new node anywhere in memory and updating one pointer. No moving, no resizing, no waste.

6. Basic Operations

Adding a node at the front

Create the new node. Set its next pointer to the current HEAD. Then move HEAD to point to the new node. Done in one step regardless of list size.

Adding a node at the back

Walk from HEAD to the end following next pointers until you find the node whose next is null. Set that node's next to your new node. The new node's next is null.

Searching

Start at HEAD. Check if the current node matches what you want. If yes, return it. If no, move to next. Repeat until found or null. This is called traversal.

Deleting a node

Walk to the node just before the one you want to delete. Update its next pointer to skip over the target and point to the target's next. Then free the memory of the deleted node.

Deleting a node means redirecting one pointer. Node B is bypassed and then freed from memory.

7. Why Everything in SENTINEL Is Built on This

Every data structure in SENTINEL, the Stack, the Queue, the Heap, the AVL Tree, the HashMap, the Graph, is built using nodes and pointers. No fixed arrays anywhere.

This means:

  • The system can grow as new patients are added without any size limit declared upfront
  • Memory is used only for what actually exists, not reserved empty slots
  • Each structure can grow and shrink dynamically as the epidemic evolves
The linked list is not one of the seven data structures in SENTINEL. It is the material everything else is made of. Like how concrete is not a building, but every building is made of it.
