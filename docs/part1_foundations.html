Foundations: Memory, Nodes, and Linked Lists

Before any data structure makes sense, you need to understand how data actually lives in memory. This part builds that base completely from scratch, because everything else in SENTINEL depends on it.

How Memory Works

Your computer's RAM is like a very long street with millions of numbered houses. Each house holds one small piece of data. The house number is called an address.

When your program needs to store something, the operating system hands it some houses to use. The question is: which houses, and how do you find your data again later?

RAM is a long row of numbered slots. Each slot holds data. Your program accesses data by its address.

Arrays: The Simple Way

The simplest way to store a list of things is an array. You ask for some boxes in a row, the OS gives you consecutive addresses, and you use index numbers to find each item instantly.

An array: five consecutive boxes. Access any element instantly using its index number.

The problem with arrays: You must decide the size upfront. If you ask for 5 boxes and later need a 6th, you cannot simply add one. The next slot in memory might already be occupied by something else. You would have to copy everything to a new, larger location. For a live epidemic system that can receive new patients at any time, this is unworkable.

Pointers: The Key Idea

A pointer is a variable that does not hold data. It holds an address. It points to where the actual data lives.

Think of it this way. Instead of writing Ahmed's phone number on a piece of paper, you write "Ahmed's number is in the red notebook on page 4." That piece of paper is a pointer. It does not contain the number, it tells you where to find it.

A pointer holds an address. Follow the address and you reach the real data.

In C++, you write a pointer like this: int* ptr = &someVariable;. The * means "this is a pointer." The & means "give me the address of."

Nodes: Data Plus a Pointer

Now combine the two ideas. What if each piece of data also held a pointer to the next piece of data? You would not need consecutive memory at all. Each item knows where the next item is.

This combined thing, data plus a pointer, is called a node. It is the single most important building block in every data structure inside SENTINEL.

A node is data and a pointer living together. The pointer says exactly where the next node is.

In C++ a node looks like this:

struct Node {
  string name;
  int age;
  Node* next; // pointer to the next node
};

Node* next is a pointer to another Node. That is the entire idea. This one pattern is the foundation every structure in SENTINEL builds on.

Linked List: Chaining Nodes Together

A linked list is simply a chain of nodes. The first node points to the second. The second points to the third. The last one points to nullptr, which means nothing, end of the chain.

You only need to hold onto the first node, called the HEAD. From there you can reach every other node by following the pointers.

A linked list. HEAD points to the first node. Each node points to the next. The last node points to null signaling the end of the chain.

Why linked lists beat arrays for most of SENTINEL: Nodes can live anywhere in memory. No consecutive slots needed. Adding a new patient means creating one new node and updating one pointer. No copying, no resizing, no wasted reserved space. The system grows naturally as the outbreak grows. Arrays still win when you need index-based math, which is exactly why the Heap uses one.

Basic Operations

Adding at the front

Create the new node. Set its next pointer to the current HEAD. Move HEAD to point to the new node. One step, regardless of list size.

Adding at the back

Walk from HEAD following next pointers until you find the node whose next is null. That is the last node. Set its next to your new node. The new node's next is null.

Searching

Start at HEAD. Check if the current node matches what you want. If yes, return it. If no, move to next. Repeat until found or null. This walking process is called traversal.

Deleting a node

Walk to the node just before the one you want to delete. Update its next pointer to skip over the target and point directly to the target's next. Then free the deleted node's memory.

Deleting a node means redirecting one pointer. Node B is bypassed, then its memory is freed.

Why Everything in SENTINEL Is Built on This

Almost every data structure in SENTINEL is built node by node using this pattern. The Stack, Queue, AVL Tree, HashMap, and Graph all use linked nodes with pointers. The Heap is the exception: it uses a fixed-size array with index arithmetic to navigate parent and child positions.

For the linked-list-based structures, this means:

  • The system can grow as new patients arrive without any size declared upfront
  • Memory is used only for what actually exists, never reserved and wasted
  • Each structure shrinks and grows dynamically as the outbreak evolves
The linked list is the foundation most structures in SENTINEL are built on. The Stack, Queue, AVL Tree, HashMap, and Graph all use nodes and pointers. The one exception is the Heap, which uses a fixed array with parent/child index arithmetic instead. Understanding both approaches matters: pointers give you dynamic growth, arrays give you O(log n) index math that makes heapify possible.
